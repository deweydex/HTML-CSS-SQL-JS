What is HTML?

HTML (HyperText Markup Language) is the standard language for creating web pages. It uses tags to structure content.

HTML Structure Example:

<!DOCTYPE html>
<html>
<head>
    <title>My Page</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is a paragraph.</p>
</body>
</html>

Common HTML Tags:

  • <h1> to <h6> - Headings (h1 largest, h6 smallest)
  • <p> - Paragraph
  • <a href="url"> - Link
  • <img src="image.jpg" alt="description"> - Image
  • <div> - Container/division
  • <span> - Inline container
  • <ul> and <li> - Unordered list
  • <table>, <tr>, <td> - Table elements

What is CSS?

CSS (Cascading Style Sheets) controls how HTML elements look on the page - colors, fonts, layouts, and more.

Three Ways to Add CSS:

  1. Inline: <p style="color: blue;">Text</p>
  2. Internal: In <style> tags within <head>
  3. External: Separate .css file (Best practice!)

CSS Syntax:

selector {
    property: value;
    another-property: value;
}

/* Example: */
p {
    color: blue;
    font-size: 16px;
    margin: 10px;
}

CSS Selectors

  • element - Selects all elements of that type (e.g., p)
  • .class - Selects elements with that class (e.g., .highlight)
  • #id - Selects element with that ID (e.g., #header)
  • element.class - Combines selectors (e.g., p.intro)

üéØ Teaching Tips - SQL Section

üí° Recommended Approach
  • Start with SELECT *: Let students see all data before filtering
  • Build complexity gradually: SELECT ‚Üí WHERE ‚Üí ORDER BY ‚Üí Aggregate functions

‚ö†Ô∏è Common Student Mistakes
  • Forgetting semicolons: Not required in this browser environment, but good practice
  • UPDATE/DELETE without WHERE: "I updated ALL the records!" - Teach paranoia
  • Comparison operators: Confusing = (equals) with == or !=
  • String quotes: SQL uses single quotes for strings: 'Alice' not "Alice"
  • Column names in WHERE: Forgetting table columns exist: WHERE age > 18 not WHERE student.age > 18
  • ORDER BY direction: Forgetting ASC/DESC (defaults to ASC)

What is SQL?

SQL (Structured Query Language) is used to manage and query databases. It lets you store, retrieve, update, and delete data.

Basic SQL Commands

SELECT - Retrieve Data

SELECT column1, column2 FROM table_name;

Gets data from a table. Use * to select all columns.

WHERE - Filter Results

SELECT * FROM students WHERE age > 18;

Filters results based on conditions.

ORDER BY - Sort Results

SELECT * FROM students ORDER BY name ASC;

Sorts results. Use ASC (ascending) or DESC (descending).

INSERT - Add New Data

INSERT INTO students (name, age, grade) 
VALUES ('John', 20, 85);

Adds new rows to a table.

UPDATE - Modify Data

UPDATE students SET grade = 90 
WHERE name = 'John';

Changes existing data. Always use WHERE!

DELETE - Remove Data

DELETE FROM students WHERE id = 5;

Removes rows from a table. Use carefully!

Practice Exercises

These exercises progress from simple to complex. You can:

  • Choose exercises that match your current comfort level

Exercise 1: Basic SELECT

Select only the name and age columns from the students table.

Show Hint

Use: SELECT column1, column2 FROM table_name;

‚úÖ Solution:
SELECT name, age FROM students;

Expected Result: A table showing all students' names and ages (8 rows)

Teaching Note: Emphasize that * selects all columns, but naming specific columns is more efficient and clearer.

Exercise 2: WHERE Clause

Find all students whose grade is less than 75.

Show Hint

Use the WHERE clause with the < operator.

‚úÖ Solution:
SELECT * FROM students WHERE grade < 75;

Expected Result: Frank Miller (grade 73)

Teaching Note: Discuss comparison operators: =, !=, <, >, <=, >=

Extension: Ask students to find all students with grades between 75 and 85: WHERE grade >= 75 AND grade <= 85

Exercise 3: INSERT

Add yourself as a new student to the database!

Show Hint

Use: INSERT INTO students (name, age, grade) VALUES ('Your Name', 20, 85);

‚úÖ Solution:
INSERT INTO students (name, age, grade) VALUES ('Your Name', 20, 85);

Verification: Run SELECT * FROM students; to see the new student

Teaching Note: The id column auto-increments, so students don't need to provide it.

Common Error: Students forget quotes around strings. Numbers don't need quotes, but text does!

Exercise 4: Sorting

Show all students sorted by name in alphabetical order.

Show Hint

Use ORDER BY with ASC for ascending order.

‚úÖ Solution:
SELECT * FROM students ORDER BY name ASC;

Note: ASC is default, so ORDER BY name also works

Teaching Note: Show DESC for reverse order. Discuss multiple sort keys: ORDER BY grade DESC, name ASC

Exercise 5: COUNT Function

How many courses are in the database?

Show Hint

Use: SELECT COUNT(*) FROM table_name;

‚úÖ Solution:
SELECT COUNT(*) FROM courses;

Better version with alias:

SELECT COUNT(*) AS total_courses FROM courses;

Expected Result: 5 courses

Teaching Note: Introduce other aggregate functions: SUM(), AVG(), MAX(), MIN()

üåü Bonus Exercises (Advanced Students)

Exercise 6: UPDATE

Challenge: Give all students with grades below 75 a 5-point bonus.

UPDATE students SET grade = grade + 5 WHERE grade < 75;

Verification: Check Frank Miller's grade increased from 73 to 78

Exercise 7: DELETE

Challenge: Remove all students under age 20.

DELETE FROM students WHERE age < 20;

Warning: This permanently removes data! Emphasize the importance of WHERE clause.

Exercise 8: Complex Query

Challenge: Find students aged 20 or 21 with grades above 85, sorted by grade descending.

SELECT * FROM students 
WHERE (age = 20 OR age = 21) AND grade > 85 
ORDER BY grade DESC;

Expected Result: Henry Moore (94), Eve Davis (91), Grace Wilson (89)

Exercise 9: Aggregate with WHERE

Challenge: What's the average grade of students aged 20?

SELECT AVG(grade) AS average_grade FROM students WHERE age = 20;

Expected Result: Average of Alice (88), David (85), Henry (94) = 89

Exercise 10: Multiple Tables

Challenge: Find all 4-credit courses.

SELECT * FROM courses WHERE credits = 4;

Expected Result: Introduction to Programming, Database Systems

üîß Troubleshooting Common Issues
  • Database won't initialize: Check browser console. Usually a CDN loading issue. Have students refresh.
  • "My SQL query doesn't work!" Check for: missing quotes around strings, typos in table/column names, missing WHERE in UPDATE/DELETE.
  • CSS not applying: Check selector specificity. Use DevTools to see which rules are being overridden.
  • Form won't submit: preventDefault() is used for demo purposes. Explain that real forms send data to servers.
